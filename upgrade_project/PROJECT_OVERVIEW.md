## 技术栈决策

### 后端
- **语言**：Python 3.11+
- **Web框架**：FastAPI（保持）
- **认证**：JWT + PyJWT
- **ORM**：SQLAlchemy 2.0 + Alembic
- **缓存**：Redis + redis-py
- **消息队列**：Redis Stream / Celery
- **向量库**：Qdrant（本地部署）

### 前端
- **框架**：Vue 3 + TypeScript
- **构建**：Vite
- **状态管理**：Pinia
- **UI组件**：Element Plus
- **图表**：ECharts
- **3D可视化**：Three.js（可选）

### 数据库
- **关系型**：PostgreSQL 16
- **向量库**：Qdrant
- **缓存**：Redis 7
- **时序**：可选 InfluxDB / TimescaleDB

### DevOps
- **容器**：Docker + Docker Compose
- **编排**：Kubernetes（生产级）
- **CI/CD**：GitHub Actions
- **监控**：Prometheus + Grafana
- **日志**：Loki + Grafana
- **追踪**：OpenTelemetry + Jaeger

## 开发规范

### 代码规范
- 严格遵循 PEP 8
- 全量类型注解（Type Hints）
- Google 风格文档字符串
- 每个函数/类必须有 docstring

### 测试规范
- 单元测试覆盖率 >= 80%
- 核心模块覆盖率 >= 90%
- 每个 PR 必须通过全部测试
- 新增功能必须附带测试

### 安全规范
- 所有用户输入必须验证
- 敏感数据加密存储
- 安全关键操作必须审计日志
- 定期安全扫描与渗透测试

### 文档规范
- 每个模块有 README
- API 自动生成 Swagger 文档
- 架构决策记录（ADR）
- 变更日志（CHANGELOG）
